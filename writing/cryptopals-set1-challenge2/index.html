Cryptopals - Set 1 Challenge 2: Fixed XOR

February 19, 2022 | Suggest Changes

This post explains how to solve the second cryptopals challenge, how to produce the fixed XOR combination of two strings in Go.

Fixed XOR#

The second cryptopals challenge states:

Write a function that takes two equal-length buffers and produces their XOR combination.

If your function works properly, then when you feed it the string:

1c0111001f010100061a024b53535009181c

… after hex decoding, and when XOR’d against:

686974207468652062756c6c277320657965

… should produce:

746865206b696420646f6e277420706c6179

The Basics#

This challenge requires you to understand XOR, which is an operator used to compare two bits (i.e. 0 and 1).

It returns 1 if the two bits are different, and 0 otherwise. Here are all the possible results of XOR-ing two bits:

  | bit1 | 0  | 0  | 1  | 1 |
  | ---  | -- | -- | -- | --|
  | bit2 | 0  | 1  | 0  | 1 |
  | ---  | -- | -- | -- | --|
  | XOR  | 0  | 1  | 1  | 0 |

In order to XOR two numbers, we need to:

  1. Check that they are the same length
  2. Represent each value in binary
  3. Line them up, and XOR each digit

Solution#

The challenge asks us to XOR these two hex values:

1c0111001f010100061a024b53535009181c

686974207468652062756c6c27732065796

We can use our HexToBytes function from challenge 1, and convert each hex to bytes and decimal.

This will give us:

  // 1c0111001f010100061a024b53535009181c in bytes
  [ 28   1  17  0  31   1   1  0  6  26   2  75 83  83 80   9  24  28]

  // 686974207468652062756c6c277320657965 in bytes
  [104 105 116 32 116 104 101 32 98 117 108 108 39 115 32 101 121 101]

Then, we convert each number to binary, and XOR the bits.

For example, the first ones are 28 and 104, which map to:

  |  28 in binary | 0 | 0 | 0 | 1 | 1 | 1 | 0 | 0 |
  | ------------- | - | - | - | - | - | - | - | - |
  | 104 in binary | 0 | 1 | 1 | 0 | 1 | 0 | 0 | 0 |
  | ------------- | - | - | - | - | - | - | - | - |
  | XOR in binary | 0 | 1 | 1 | 1 | 0 | 1 | 0 | 0 |

The XOR value of the first byte, 01110100, is 116.

Once we have done this for byte, we hex encode the output to get the answer.

Solution in Go#

The programmatic solution is below.

^ is the bitwise XOR operator in Go.

Note that set1.HexToBytes and set1.BytesToHex were implemened in Set1 Challenge1.

package main

import (
    "fmt"
    "log"

    "github.com/julieqiu/cryptopals/set1"
)

const (
    input1 = "1c0111001f010100061a024b53535009181c"
    input2 = "686974207468652062756c6c277320657965"
)

func main() {
    b1, err := set1.HexToBytes(input1)
    if err != nil {
        log.Fatal(err)
    }
    b2, err := set1.HexToBytes(input2)
    if err != nil {
        log.Fatal(err)
    }

    // Confirm the inputs are the same length before XOR-ing.
    if len(b1) != len(b2) {
        log.Fatalf("len(b1) = %d and len(b2) = %d", len(b1), len(b2))
    }

    var out []byte
    for i := 0; i < len(b1); i++ {
        out = append(out, b1[i]^b2[i])
    }
    fmt.Println(bytesToHex(out))
}

// bytesToHex converts a byte slice to the equivalent in hex.
//
// For use in future challenges in this series, this function is at
// https://pkg.go.dev/github.com/julieqiu/cryptopals/set1#BytesToHex.
func bytesToHex(src []byte) string {
    return hex.EncodeToString(src)
}
